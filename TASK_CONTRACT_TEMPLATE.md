# TASK_CONTRACT_TEMPLATE.md

Use this for any autonomous execution request.

## 1) Goal
One concrete outcome.

## 2) Track
Exactly one: `SMB` / `UPSKILL` / `BRAND` / `LIFE` / `SYSTEM`

## 3) Inputs
- Required files
- Required sources/URLs
- Required scripts/tools

## 4) Constraints
- Timebox
- Budget/model lane constraints
- Security/approval boundaries

## 5) Deliverables
- Exact files to create/update
- Exact summary format required

## 6) Done Checks
- Objective pass criteria
- Validation commands/tests
- For research tasks: relevance and entrepreneur-importance thresholds from `config/research_signal_policy.json`

## 7) Fail State
`BLOCKED: missing = [...] | fastest path = [...]`

## 8) Response Format
Must follow `OUTPUT_STANDARD.md`.
